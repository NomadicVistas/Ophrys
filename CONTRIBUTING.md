# Contributing to Ophrys

Ophrys welcomes focused experiments that strengthen the artistic proposition and public accountability of the installation.

## Before proposing a contribution

Read:

- [the hackathon build brief](docs/HACKATHON.md);
- [the prototype architecture](docs/ARCHITECTURE.md);
- [the ethics boundary](docs/ETHICS.md).

## Good first contributions

- a small spatial signal study;
- a privacy-preserving observation component;
- a transparent adaptation policy;
- a public state and uncertainty display;
- a refusal mechanism with downstream consequences;
- documentation of energy, hardware and network dependencies.

Use an issue to describe the proposition, required hardware, data handled, expected public experience and how failure will be represented.

## Non-negotiable exclusions

Contributions must not introduce facial recognition, demographic inference, emotion classification, covert profiling, compelled disclosure or hidden persuasive optimisation.

## Licensing

Contributions are accepted under the repository's MIT Licence unless explicitly agreed otherwise before submission.
